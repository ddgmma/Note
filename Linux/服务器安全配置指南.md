## 👤 用户权限配置

### 创建普通用户
```bash
# 创建用户并授予sudo权限
adduser ddgmmf
usermod -aG sudo ddgmmf
```

---
## 配置用户SSH密钥
```bash
# 创建.ssh目录并设置密钥
mkdir -p /home/ddgmmf/.ssh
cp /root/.ssh/authorized_keys /home/ddgmmf/.ssh/

# 设置严格权限
chmod 700 /home/ddgmmf/.ssh
chmod 600 /home/ddgmmf/.ssh/authorized_keys
chown -R ddgmmf:ddgmmf /home/ddgmmf/.ssh
```

- **控制台路径**: ECS → 网络与安全 → 密钥对
- **操作**: 创建密钥对 → 自动新建 → 下载.pem私钥

### 密钥文件存放位置

```text
C:\Users\ddgmma\Desktop\File\aliyun_server.pem
```
### 连接命令 ###
```bash

ssh -i "C:\Users\ddgmma\Desktop\File\aliyun_server.pem" ddgmmf@ddgmm.top

# 连接root用户（已禁用）
ssh -i "C:\Users\ddgmma\Desktop\File\aliyun_server.pem" root@ddgmm.top
```

---
## 🔒 SSH安全配置
### 修改SSH配置 ###
```bash
sudo vim /etc/ssh/ssh/sshd_config
```
## 关键配置项
```text
PermitRootLogin no              # 禁用root SSH登录
PasswordAuthentication no       # 禁用密码登录
ChallengeResponseAuthentication no
UsePAM no
```
## 重启SSH服务
```bash
sudo systemctl restart ssh
```

---
## 🛡️权限说明
### 权限命令
```bash
# 设置文件权限：仅所有者可读写
chmod 600 /home/ddgmmf/.ssh/authorized_keys

# 递归更改所有者
chown -R ddgmmf:ddgmmf /home/ddgmmf/.ssh
```
### 权限规则
-**600**:用户:读写(4+2),组:无,其他:无
-**700**:用户:读写(4+2+1),组:无,其他,:无

## 🆘 紧急恢复方案
### 被锁在外面怎么办?

1.阿里云控制台→示例→远程连接→VNC

2.通过VNC直接代理root账户

3.修复SSH配置:

```bash
vim /etc/ssh/sshd_config
# 改回：PermitRootLogin yes
systemctl restart ssh
```

## 💡 重要提醒

### 安全注意事项

-备份私钥：私钥丢失无法恢复

-权限严格：SSH对密钥文件权限要求苛刻

-测试再禁用：确保新用户能登录后再禁用root

-保留VNC：永远保留控制台访问作为备用

 ### 系统优化建议

```bash
# 首次登录后执行
sudo apt update && sudo apt upgrade -y
sudo apt install -y vim git curl wget htop
```
